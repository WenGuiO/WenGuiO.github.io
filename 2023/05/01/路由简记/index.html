路由简记

RIP（Routing Information Protocol）

协议介绍

一种基于距离矢量算法来计算到达目的网络的最佳路径路由协议，它通过UDP报文进行路由信息的交换，使用的端口号为520，RIP是基于跳数（最多支持15跳）来衡量到达目的地址的距离，称为度量值。

工作流程

工作方式：每隔一定时间，每台路由器都会向它的邻居路由器发送自己所知道的所有路由信息，并接收邻居路由器发送的路由信息。在接收到邻居路由器发送的信息后，路由器会将新的路由信息与自己已有的路由信息进行比较，并更新自己的路由表。

  1. 启动
    当一个路由器启动时，会向相连的邻居节点发送RIP路由更新消息。这些消息包含该路由器的路由表中所有可达目的地和到达这些目的地所需的跳数信息。

  2. 定期更新
    每个路由器会定期发送RIP路由更新消息到邻居节点路由器。默认情况下，更新间隔为30秒。

  3. 路由信息处理
    每个路由器接收到RIP路由更新消息后，会进行处理，更新本地路由表。

  • 如果接收消息是来自已知路由器的更新，则会使用该信息更新已知路由器的路由表。
  • 如果接收消息是来自未知路由器的更新，则会将该路由器的信息添加到路由表中，并使用该信息计算到达其他目的地所需的跳数。

  1. 路由表维护
    每个路由器都会维护自己的路由表，该路由表包含到达网络中每个目的地所需的最小跳数。路由器会定期（30s）发送自己的路由表到网络中的所有其他路由器，并使用接收到的路由表信息更新自己的路由表。

  2. 路由表失效与删除

失效：如果路由器在180s内未收到其他路由器的更新消息，就会认为与该路由器相连的链路已经失效，即metric值标记为16。

删除：如果在标记失效后的120s内仍然没有接收到更新信息，就会删除该路由。

算法实现

RIP协议使用距离向量算法–贝尔曼-福德算法（Bellman-Ford Algorithm）实现。

运作过程：

  1. 初始化路由表
    首先，每个节点初始化自己的路由表。路由表中包含可达的目的地及其到达该目的地的距离和下一跳节点的信息。

    初始化时，本地节点的距离为0，下一跳节点为NULL；

    对于其他目的地，则将其距离设为无穷大，下一跳节点为NULL。

  2. 发送路由信息
    每个节点定期地向它的邻居节点发送路由信息。

    发送的路由信息包括本地节点的所有目的地和距离。

  3. 接收路由信息
    当节点收到来自邻居节点的路由信息时，它会根据这些信息更新自己的路由表。

    如果路由表中不存在某个目的地，就将其添加到路由表中；

    如果目的地已经存在于路由表中，就存储距离小的路由。

  4. 更新路由表
    当节点收到来自邻居节点的路由信息后，可能需要更新自己的路由表。

    如果有目的地的距离发生了改变，则节点需要重新计算它到每个目的地的距离，并更新路由表中的信息。这个过程会一直进行，直到路由表收敛，所有节点的路由表中的信息都不再发生改变。

  5. 触发更新
    如果路由表中的某个目的地的距离发生了改变，则节点需要通知它的邻居节点，以便它们可以更新自己的路由表。

特点总结

  1. 简单：使用跳数作为距离度量标准，实现简单。
  2. 易部署：只需在每个路由器上启用RIP协议即可使用。
  3. 只适用于小型网络。
  4. 有限计数：RIP协议规定最大跳数为15，也限制了该协议的使用范围。

OSPF（Open Shortest Path First）

协议介绍

基于链路状态（Link State）的路由选择算法。在OSPF网络中，每个节点都会维护一张完整的网络拓扑图，并通过周期性的交换链路状态信息来保持拓扑图的一致性。节点之间通过计算链路的度量值（如带宽、延迟等）来选择最优路径，并利用该路径进行数据包的转发。

工作流程

OSPF协议的工作流程可以分为邻居发现、链路状态信息交换、网络拓扑图的构建、路由计算和数据包转发阶段。

  1. 邻居发现：每个OSPF节点会向相邻节点发送Hello报文，用于建立邻居关系和交换路由信息。在Hello报文中，节点会携带自己的信息（如IP地址、路由器ID等），并要求相邻节点返回相同的信息。当两个节点交换到彼此的信息后，就建立了邻居关系。

  2. 信息交换：节点之间周期性地交换链路状态信息（LSA），用于构建网络拓扑图。当节点收到LSA时，会对其进行解析，从而获得关于网络拓扑图的信息，如邻居列表、链路状态和度量值等。

  3. 构建网络拓扑图：每个节点通过接收并解析其他节点发送的LSA，建立起一张完整的网络拓扑图，并计算出到各个目的地的最短路径。在构建网络拓扑图的过程中，节点会使用Dijkstra算法或其变种来计算最短路径。

  4. 路由计算：节点根据收集到的链路状态信息，计算出到各个目的地的最短路径，并将其记录在自己的路由表中。路由表中通常包括目的网络地址、下一跳地址和度量值等信息。

  5. 数据包转发：节点根据自己的路由表，选择最短路径来转发数据包。当节点收到一个数据包时，它会查找自己的路由表，找到到达目的地的最短路径，并将数据包沿着该路径转发出去。

  6. 故障检测和恢复：如果某个链路发生故障，节点会向相邻节点发送通知，从而触发链路状态信息的更新和路由计算的重新进行。节点会重新计算最短路径，并更新自己的路由表。这样，当数据包到达该节点时，它会选择新的最短路径进行转发。

算法实现

OSPF协议使用的是Dijkstra算法或其变种，用于计算最短路径。具体来说，OSPF协议使用了以下的算法实现：

  • Dijkstra算法：一种基于贪心策略的最短路径计算算法。在OSPF协议中，每个节点都维护着一个链路状态数据库（LSDB），其中包含了整个网络拓扑图的链路状态信息。每个节点会从自己的LSDB中选取距离自己最近的节点，并将其加入到自己的最短路径树中。然后，节点会更新自己的距离向量，并继续寻找下一个距离自己最近的节点，直到所有的节点都被加入到最短路径树中。这样，就可以得到到各个目的地的最短路径。
  • SPF（Shortest Path First）算法：Dijkstra算法的一种优化算法，用于加速最短路径计算的过程。在OSPF协议中，每个节点都会周期性地执行SPF算法，以更新自己的路由表。SPF算法首先会选取距离自己最近的节点，并将其加入到自己的最短路径树中。然后，它会检查与该节点相邻的所有节点，并计算到这些节点的距离。如果新的距离比原来的距离更短，则会更新路由表，并将相邻节点加入到候选列表中。这样，SPF算法可以通过动态调整候选列表，逐步扩展最短路径树，并得到到各个目的地的最短路径。
